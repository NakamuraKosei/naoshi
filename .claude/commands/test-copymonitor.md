# /test-copymonitor — Naoshi自動テストループ

GPTでAIレポートを生成 → Naoshi本番で変換 → CopyMonitorでスコア検証 → 結果を蓄積するフルサイクル自動テスト。

## 引数
- `$ARGUMENTS` にテーマを指定可能（例: `/test-copymonitor SNSと若者のメンタルヘルス`）
- テーマ未指定の場合はランダムに選択

## 実行手順

### 前提条件
以下のタブがPlaywrightで開いていること（初回は手動セットアップ）:
- ChatGPT (https://chatgpt.com) — ログイン不要でもOK
- Naoshi (https://naoshi.vercel.app/app) — ログイン済み
- CopyMonitor (https://www.copymonitor.jp/gptkiller/upload) — ログイン済み

### Step 1: テーマ決定
`$ARGUMENTS` があればそれを使う。なければ以下からランダムに1つ選ぶ:
文系:
- SNSが若者のメンタルヘルスに与える影響
- 大学教育におけるAI活用の是非
- リモートワークが労働生産性に与える影響
- ジェンダー平等に向けた日本の課題
- 食品ロス削減に向けた法的アプローチ
- 若者の政治参加とソーシャルメディアの役割
- フェイクニュースの拡散メカニズムと対策
- 移民政策と多文化共生の課題
理系:
- 再生可能エネルギーの効率と技術的課題
- 抗生物質耐性菌の現状と対策
- 気候変動モデルの精度と限界
- プラスチックの生分解技術と環境への応用
- 量子コンピュータの原理と実用化への課題
- 水資源の枯渇問題と淡水化技術
- 遺伝子編集技術CRISPRの医療応用と倫理

### Step 2: ChatGPTでレポート生成
1. Playwrightで ChatGPT タブに切り替え
2. テキストボックスに以下のプロンプトを入力して送信:
```
以下の条件で日本語の大学レポートを書いてください。

テーマ: {選んだテーマ}
文字数: 2000〜3000字
文体: だ・である調
構成: 序論・本論・結論の3部構成
参考文献: 架空でよいので3つ付けてください
注意: 大学2年生が書くレベルの学術レポートとして自然なものにしてください。
```
3. 生成完了を待機（「停止」ボタンが消えるまで）
4. `[data-message-author-role="assistant"]` の最後の要素からテキスト取得
5. テキストをファイルに保存: `test-results/{timestamp}_input.txt`

### Step 3: Naoshiで変換
1. Naoshi タブに切り替え
2. 入力テキストエリアにStep 2のテキストを貼り付け
3. 「だ・である調」を選択
4. 「なおす」ボタンをクリック
5. 変換完了を待機（「変換中…」が消えるまで、最大120秒）
6. 出力テキストエリアから変換結果を取得
7. 変換結果をファイルに保存: `test-results/{timestamp}_output.txt`

### Step 4: CopyMonitorでチェック
1. CopyMonitor タブに切り替え
2. 「新規アップロード」ページに移動（必要なら）
3. 「テキスト入力」ボタンをクリック
4. タイトル欄にテーマ名を入力
5. 内容欄にStep 3の変換結果を貼り付け
6. 「保存」をクリック
7. 「チェック」をクリック
8. 文書種類ダイアログで「課題、レポート」を選択
9. ダイアログ内の「チェック」をクリック
10. 結果リストページで最新エントリのスコアを読み取る

### Step 5: 結果記録
1. `test-results/results.jsonl` に1行追加（JSON Lines形式）:
```json
{
  "timestamp": "2026-04-16T22:10:00+09:00",
  "theme": "テーマ名",
  "input_chars": 1449,
  "output_chars": 1346,
  "score": 100,
  "prompt_version": "evasion-v1.1",
  "input_file": "20260416_221000_input.txt",
  "output_file": "20260416_221000_output.txt"
}
```
2. VOICEVOXで結果を音声通知:「テスト完了。コピーモニタースコアは{score}パーセントです。」
3. 結果サマリーをテキスト出力

### Step 6: 繰り返し判定
- ユーザーに「続けますか？」と確認
- 続ける場合はStep 1に戻る（別テーマ）
- 終了する場合は蓄積データのサマリーを表示

## 結果ファイル
- `test-results/results.jsonl` — 全テスト結果（JSON Lines）
- `test-results/{timestamp}_input.txt` — 各テストの入力テキスト
- `test-results/{timestamp}_output.txt` — 各テストの出力テキスト

## データ分析コマンド
蓄積後に使えるクエリ例:
- 平均スコア
- テーマ別スコア分布
- 文字数とスコアの相関
- プロンプトバージョン間の比較
