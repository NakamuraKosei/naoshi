## 1. デザインコンセプト

### 1.1 キーワード
**「清潔・信頼・学生にやさしい」**

白を基調にブルーで差し色を入れ、学術ツールとしての信頼感と、学生向けサービスとしての親しみを両立させる。Sidekicker.ai のような海外系ヒューマナイザーの「余白を大きく取ったミニマル構成」を参考にしつつ、日本語学術文の読みやすさを優先する。

### 1.2 参考にしたサイト
- **Sidekicker.ai**: 構成・余白感・セクションの交互パターン
- **Linear / Vercel / Notion**: 白ベースのミニマル感
- **STUDIO / note**: 日本語UIのやわらかさ

### 1.3 雰囲気
- ミニマルでプロフェッショナル
- 派手な装飾や影は極力使わない
- 大きめの余白で呼吸感のあるレイアウト

## 4. スペーシング（Sidekicker基準で大きめ）

4pxベースのグリッド。Sidekickerを参考にセクション間を大きく取る。

| Token | px | 用途 |
|---|---|---|
| `space-1` | 4 | アイコンとテキストの隙間 |
| `space-2` | 8 | 小さな要素の間 |
| `space-4` | 16 | デフォルトパディング |
| `space-6` | 24 | カード内側 |
| `space-8` | 32 | セクション内の要素間 |
| `space-12` | 48 | サブセクション間 |
| `space-16` | 64 | セクション内パディング（縦） |
| `space-24` | 96 | セクション間（PC） |
| `space-32` | 128 | ヒーローの縦パディング |

### 4.1 セクションの縦パディング
- モバイル: `py-16`（64px）
- タブレット: `py-20`（80px）
- デスクトップ: `py-24`（96px）〜 `py-32`（128px、ヒーローのみ）

### 4.2 コンテンツ最大幅
- LP本文: `max-w-6xl`（1152px）を中央寄せ
- ヒーローのタイトル部分: `max-w-3xl`
- 変換画面: `max-w-7xl`（1280px）

## 5. ボーダー・影・角丸

### 5.1 角丸
| Token | px | 用途 |
|---|---|---|
| `radius-sm` | 6 | タグ |
| `radius-md` | 10 | ボタン、入力欄 |
| `radius-lg` | 14 | カード |
| `radius-xl` | 20 | 大きなカード、モーダル |
| `radius-full` | 9999 | ピル型ボタン、アバター |

### 5.2 影（控えめに）
- `shadow-sm`: `0 1px 2px rgba(15, 23, 42, 0.04)` — カードのデフォルト
- `shadow-md`: `0 4px 16px rgba(15, 23, 42, 0.06)` — ホバー時
- `shadow-lg`: `0 16px 40px rgba(15, 23, 42, 0.10)` — モーダル

Sidekickerのように、影は極力控えめにしてボーダーで要素を区切る方針。

### 5.3 ボーダー
- デフォルト: `1px solid #E5E7EB`
- フォーカスリング: `2px solid #2563EB`、`outline-offset: 2px`

## 7. 画面レイアウト詳細

### 7.1 ランディングページ（/）

Sidekickerの構成を踏襲し、以下の順で積む：

#### 7.1.1 ヒーロー（白背景）
- 縦パディング: `py-24` 〜 `py-32`
- 中央寄せ、最大幅 `max-w-3xl`
- 構成:
  1. Display見出し（56px）: 「AIで書いたレポートを、ちゃんと人間の言葉に。」
  2. サブコピー（Body Large）: サービスの1〜2行説明
  3. 評価行（将来）: 星 + 「利用者満足度 4.9/5」
  4. Primaryボタン「無料ではじめる」+ Secondaryボタン「使い方を見る」
  5. 下部に変換画面のスクリーンショット or イメージ

#### 7.1.2 特徴セクション（Primary Lighter背景）
- 縦パディング: `py-24`
- 3カラム（モバイル縦積み）
- 各カード:
  - アイコン（Lucide、ストローク2px、色はPrimary）
  - H3見出し
  - Body Small 説明文
- 特徴例:
  1. 日本語学術文に特化
  2. 参考文献を完全保護
  3. だ・である調 / ですます調を選べる

#### 7.1.3 使い方セクション（白背景）
- 縦パディング: `py-24`
- 3ステップ横並び（モバイル縦積み）
  1. 貼り付ける
  2. なおすボタンを押す
  3. コピーして使う

#### 7.1.4 料金セクション（Primary Lighter背景）
- 縦パディング: `py-24`
- 見出し: 「シンプルな料金体系」
- 4プランカード横並び（タブレット2x2、モバイル縦積み）
- ヘビー月額にのみ「人気」バッジ

#### 7.1.5 FAQ（白背景）
- 縦パディング: `py-24`
- 最大幅: `max-w-3xl`
- アコーディオン形式

#### 7.1.6 CTAバナー（ダーク `#0F172A` 背景）
- 縦パディング: `py-20`
- 白文字の大見出し + 白ボタン
- 「今日からレポートを整えよう」等

#### 7.1.7 フッター（白背景）
- 縦パディング: `py-12`
- ロゴ / リンク（利用規約・プライバシー・特商法・お問い合わせ）/ コピーライト

### 7.2 変換画面（/app）
- ヘッダー: 白背景、下ボーダー、高さ64px、ロゴ + 文体セレクタ + アカウントアイコン
- メイン: 最大幅 `max-w-7xl`、`py-8`

  - 左: 入力カード（`min-h-[560px]`、下に文字数）
  - 右: 出力カード（同じ高さ、下にコピーボタン）
- 中央下部: Primaryボタン「なおす」（幅広め、`py-4`）

### 7.3 料金ページ（/pricing）
- LPの料金セクションを拡張した構成
- 各プランに詳細な機能リスト

### 7.4 アカウント（/account）
- 上部にプラン状態カード（大）
- 下部に使用回数カード + Customer Portalリンク
- 2カラム（PC）／縦積み（モバイル）

### 7.5 ログイン（/login）
- 背景: `#F5F9FF`
- 中央に白カード（最大幅 `max-w-md`、パディング `p-10`、角丸 `rounded-xl`）

## 9. トーン＆ボイス（UIコピー）

### 9.1 サービス名・タグライン
- 正式名: **Naoshi（ナオシ）**
- タグライン: 「AIで書いたレポートを、ちゃんと人間の言葉に。」
- サブコピー例: 「日本の大学生のための、日本語特化ヒューマナイザー。」

### 9.2 ボタン文言
| アクション | 文言 |
|---|---|
| 変換実行 | 「なおす」 |
| コピー | 「コピー」 |
| 登録 | 「無料ではじめる」 |
| ログイン | 「ログイン」 |
| プラン変更 | 「プランを変更する」 |

- エラー: 「うまく変換できませんでした。もう一度お試しください。」

## 10. レスポンシブ方針

- `lg`: 1024px
- `xl`: 1280px

## 11. アクセシビリティ

- コントラスト比: 本文4.5:1以上（WCAG AA）
- フォーカスリング: すべてのインタラクティブ要素に
- キーボード操作: Tab で全機能アクセス可能
- `prefers-reduced-motion` に対応

---

## 12. アニメーション

### 12.1 方針
- Sidekickerのように控えめに
- 200〜300msのトランジションで十分

### 12.2 具体例
- ボタンホバー: `transition: all 200ms ease`、軽い上方向移動
- カードホバー: 影が少し濃くなる + 軽い上方向移動
- モーダル: フェード＋軽いスケール（200ms）
- スクロール演出: 必要最小限、派手なパララックスは使わない
